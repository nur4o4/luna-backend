# Output Visualizations Guide

This directory contains 4 visualizations generated by `python3 run_simulation.py`.

## Files

### 1. `group_selection_process.png`
**What it shows**: How the algorithm selects the optimal group from your social network

**Three panels**:
- **Panel 1 (Left)**: Your position in the full social graph
  - Orange = You
  - Grey = Everyone else
  - Shows social context

- **Panel 2 (Center)**: Candidate pool (friends + friends-of-friends)
  - Teal = Direct friends (algorithm considers these first)
  - Light teal = Friends-of-friends (expanded pool)
  - Grey = Outside reach
  - Shows how algorithm *intelligently limits* search space

- **Panel 3 (Right)**: The selected optimal group
  - Gold = Selected group members
  - Orange = You
  - Edges highlight group interconnections
  - Score breakdown shows why this group wins

**Key insight**: Algorithm doesn't pick random people—it works within social proximity and filters by interest alignment.

---

### 2. `propagation_waves.png`
**What it shows**: How a single recommendation spreads through your social network over time

**Five panels** (one per wave):
- **Wave 0**: Just you (seed user)
- **Wave 1**: Your immediate network sees the recommendation
- **Waves 2-4**: Waves of engagement spreading outward
- **Wave 5+**: Extended network gradually joining

**Color coding**:
- Orange = You (seed user)
- Blue = "Interested" (passive engagement)
- Peach = "Invited" (explicit invitation)
- Green = "Going" (confirmed commitment)
- Grey = Not yet engaged

**Key insight**: Demonstrates the "flywheel effect"—social proof compounds in each wave:
- Wave 1: A few friends join
- Wave 2: Their friends see social proof, more committed
- Wave 3+: Momentum becomes self-reinforcing

This is what converts individual interest into collective action.

---

### 3. `score_comparison.png`
**What it shows**: Top 5 group recommendations with their score breakdowns

**Bar chart** comparing 5 different group options across scoring components:
- **Teal** = Shared Interests (35% weight)
- **Yellow** = Availability Overlap (35% weight)
- **Blue** = Past Success (20% weight)
- **Peach** = Social Cohesion (10% weight)
- **Green** = Venue Fit (bonus)

**Key insight**: Different groups excel at different factors. The algorithm finds the group with the best overall balance—not just highest interest overlap, but availability too.

Example read:
- Group A: High interests, perfect availability → wins
- Group B: High interests, poor availability → loses
- Group C: Medium interests, perfect availability → middle score

---

### 4. `group_deep_dive.png`
**What it shows**: Detailed breakdown of the TOP recommended group

**Six panels**:
1. **Group Members** (top-left): Who's in the winning group
   - Names, interests, availability, initiator scores

2. **Score Components** (top-right): Visual breakdown of the 0.93 score
   - Horizontal bars showing contribution of each factor
   - Total score highlighted in orange box

3. **Interest Overlap Heatmap** (bottom-left): 
   - Matrix showing shared interests across group
   - Green checkmarks = shared interest
   - Darker = more people share it

4. **Availability Overlap Heatmap** (bottom-center):
   - Matrix showing time slot overlaps
   - Best slot highlighted with red border (e.g., sun_11am)
   - Shows when everyone can coordinate

5. **Algorithm Insight Box** (bottom-right):
   - Explains why this group scores 0.93
   - Breaks down contribution of each factor
   - Makes the algorithm's logic transparent

**Key insight**: Reviewers can see *exactly why* this group was chosen—all scoring is transparent and explainable.

## How to Use These in Your Presentation

**Walkthrough sequence:**
1. Start with `group_selection_process.png` — explain the algorithm and show candidate pool
2. Show `score_comparison.png` — explain tradeoffs between groups
3. Show `group_deep_dive.png` — prove the algorithm's logic is transparent
4. Finish with `propagation_waves.png` — show the flywheel effect and momentum building

**Talking points**:
- "We're not recommending individuals, we're assembling *groups*"
- "The algorithm balances interests, availability, history, and cohesion"
- "Social proof compounds—each wave increases commitment likelihood"
- "The simulation proves this model works—groups actually form"

## Regenerating Outputs

```bash
# Regenerate all visualizations
python3 run_simulation.py

# Test the algorithm with custom scenarios
python3 -m simulation.test_groups
```

Each run uses a random seed for the social graph, so outputs will vary. To keep the same graph:
- Modify `seed=42` parameter in `run_simulation.py` → `run_simulation.py`
- Same seed = deterministic outputs for reproducibility
